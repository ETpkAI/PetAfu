# “宠物阿福” (Pet Afu) 商业化自研落地计划

## [Goal Description]
打造一款定位为**“AI 宠物健康档案管家”、“健康信息检索工具”、“百科助手”**的国民级宠物应用，全面复刻“蚂蚁阿福”的陪诊陪伴模式。系统将采用 Clean Room（净室）重构策略，不直接使用侵权开源代码，而是吸收 KatApp 的多模态感官交互理念与 Pet-Care-RAG 的权威知识库检索逻辑，从零构建合规、安全、可商业化的前后端闭环。

---

## User Review Required

> [!IMPORTANT]
> **合规红线提示**：本架构设计的所有对外文案、API 返回结果，必须强制经过 Prompt 拦截层，确保不出现“确诊为XX疾病”、“必须服用XX药物”的表述，必须替换为“高度疑似XX疾病特征”、“此类特征常见于XX问题，建议参考XX护理”。

## 核心产品架构 (复刻蚂蚁阿福模式)

### 一、 核心功能模块划分

#### 1. 宠物百科助手 (重塑“AI 问诊”降低风险)
*   **多模态 AI 咨询台**：支持文字描述、语音输入症状，以及**上传异常部位/粪便/呕吐物图片**。
*   **智能追问机制**：当用户输入信息不足时（例：“我家猫吐了”），AI 会根据预设医学逻辑追问（“是吐毛球、未消化食物，还是黄水透明液体？”）。
*   **拍照解读神器**：一键拍照识别宠物处方单、生化体检报告单，用大白话解释给主人听；支持拍猫粮狗粮配料表，分析营养成分。
*   **合规话术包装**：输出结果定位为“综合兽医学术文献检索结果”，并附加强制免责声明。

#### 2. 万能宠物档案管家 (拉升应用留存率的护城河)
*   **贯穿生命周期的数字档案**：记录宠物体重变化曲线、生日、品种特点。
*   **任务打卡与提醒**：智能生成并提醒“本月内外同驱”、“年度核心疫苗（猫三联/狂犬）接种”、“绝治手术适宜期”。
*   **健康小日记**：记录每天的精神状态、食欲、排便情况。如果 AI 监控到连续三天记录“食欲不振”，会主动触发健康预警弹窗。

#### 3. 线下医疗服务导流 (商业化变现闭环)
*   当知识库检索发现当前特征属于“高危/致命”（如频繁呕吐伴随血丝、抽搐等），AI 将停止给出居家护理建议，直接弹出**“真人兽医在线急诊”**或**“附近 24 小时宠物医院导航”**入口。这是最成熟的商业变现路径（抽成/广告费）。

## Proposed Changes (技术选型与实现架构)

### [前端架构设计 (仿 KatApp 交互与视觉)]
*   **技术栈选择**：`React Native` 或 `Flutter` (推荐 React Native，拥有极佳的动态与跨平台生态，且摄像头调用方案成熟)。
*   **核心 UI 页面**：
    *   **首页 (Dashboard)**：展示宠物虚拟形象、近期健康提醒打卡、快捷拍物按钮。
    *   **AI 咨询室 (Chatbot UI)**：类似微信对话流，支持图文混排、语音气泡，AI 回答采用 Markdown 渲染以展示严谨的列表建议。
    *   **档案库 (Profiles)**：时间轴形式展示小日记和医疗记录的图表。

### [后端与 AI 架构设计 (仿 Pet-Care-RAG 底座)]
*   **后端开发语言**：`Python` (FastAPI / Django) - 最适合与 AI 及大模型后端无缝集成。
*   **数据库**：
    *   `PostgreSQL`：存储用户体系、宠物档案、日记等关系型数据。
    *   `Milvus` / `PineCone` 或 `pgvector`：用来存储《兽医内科学》、《默克兽医手册》等专业书籍切片后的**向量数据库 (Vector DB)**。
*   **AI 大脑调度层 (核心环节)**：
    *   **多模态大模型对接**：接入 OpenAI `GPT-4o` 或 Google `Gemini 1.5 Pro`，专门用于识别用户上传的病患图片或化验单。
    *   **RAG (检索增强生成) 引擎**：使用 `LangChain` 或 `LlamaIndex` 搭建。
        *   *工作流*：用户提问 -> LLM 提取关键特征（红肿, 脱毛, 鳞屑） -> 去向量数据库检索专业医学文档库 -> 将专业文献内容合并进 Prompt -> LLM 生成针对普通人通俗易懂的“健康参考报告”。

## Verification Plan

### 敏捷开发验证步骤 (MVP 最小可行性产品)
1.  **AI 问诊链路验证**：在后端硬编码一段兽医知识，通过 API Postman 上传一张狗皮肤痒的图片，验证大模型是否能返回合规的、包含免责声明的、基于预设知识库的“参考答复”。
2.  **前端交互验证**：开发 React Native 页面，跑通“拍摄照片 -> 等待 AI 思考 -> 逐字流式打字输出 (Streaming)”的体感交互。
3.  **用户闭环验证**：测试通过创建宠物档案，录入生日后，系统能否在特定日期触发“疫苗提醒”。
